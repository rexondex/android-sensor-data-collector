# 에러 처리 및 안전성 정책

## 1. API 전송 실패 처리
- 최대 3회 재시도, 실패 시 로컬 큐(AsyncStorage)에 저장
- 다음 전송 주기에 큐 데이터 우선 재시도
- 연속 실패 시 Alert 등으로 사용자에게 안내

## 2. 센서/권한 에러
- 미지원 센서는 자동 감지 후 UI/로직에서 제외
- 위치 권한 거부 시 location 센서 비활성화

## 3. 환경변수/키 관리
- API Key는 SecureStore에 안전하게 저장, 외부 노출 방지
- 키 미입력 시 데이터 전송 차단 및 입력 유도

## 4. 타입/런타임 에러
- Zod로 응답 스키마 검증, 타입 불일치 시 예외 처리
- TypeScript로 주요 타입 명시, 암시적 any 등 방지

## 5. 테스트/디버깅
- 핵심 로직은 Jest 테스트로 검증
- 테스트 실패/컴파일 에러는 즉시 수정
